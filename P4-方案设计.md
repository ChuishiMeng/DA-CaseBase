## 二、案例数据结构设计

### 2.1 核心数据模型

```python
@dataclass
class NL2SQLCase:
    """一个案例 = 原始对 + 抽象模板 + 结构指纹 + 元信息"""
    
    # ===== 唯一标识 =====
    case_id: str                    # 案例唯一ID
    version: int = 1                # 版本号
    
    # ===== 原始自然语言 & SQL =====
    nl_original: str                # 原始NL
    sql_original: str               # 原始SQL
    
    # ===== 实体分离结果 =====
    entity_slots: List[EntitySlot]  # 实体槽位列表
    # [EntitySlot("{CITY}", "地域", "北京", ...),
    #  EntitySlot("{DATE_RANGE}", "时间", "上个月", ...)]
    
    # ===== 抽象模板 =====
    nl_template: str                # NL模板（实体替换为槽位）
    sql_template: str               # SQL模板（具体值替换为槽位）
    
    # ===== 语义向量 =====
    nl_embedding: List[float]       # 原始NL向量
    nl_template_embedding: List[float]  # 模板NL向量
    
    # ===== SQL结构指纹 =====
    sql_fingerprint: SQLStructureFingerprint
    structure_hash: str             # 结构哈希（快速索引）
    
    # ===== 查询意图 & 复杂度 =====
    query_intents: List[QueryIntent]  # 查询意图
    complexity_level: int           # 复杂度等级
    
    # ===== 质量 & 使用统计 =====
    quality_score: float            # 案例质量分(0-1)
    hit_count: int                  # 被召回次数
    success_count: int              # 成功使用次数
    
    # ===== 负例关联 =====
    confusable_case_ids: List[str]  # 易混淆案例ID
    confusion_notes: Dict[str, str] # 混淆原因说明
```

### 2.2 实体槽位设计

```python
@dataclass
class EntitySlot:
    """实体槽位：将具体值与查询模式分离"""
    slot_name: str              # 槽位名: "{CITY}", "{DATE_RANGE}"
    slot_type: str              # 类型: "地域", "时间", "数值"
    original_value: str         # 原始值: "北京", "上个月"
    normalized_value: str       # 归一化值
    mapped_column: str          # 映射列: "city", "order_date"
    mapped_table: str           # 映射表: "orders"
    constraint_type: str        # 约束类型: "EQ", "GT", "BETWEEN"
```

**示例**：
```
原始NL: "查询上个月销售额超过100万的北京客户"
实体分离:
  {CITY} → 北京 → customers.city = '北京' (EQ)
  {DATE_RANGE} → 上个月 → orders.date BETWEEN '2024-01-01' AND '2024-01-31'
  {THRESHOLD} → 100万 → 1000000 (GT)

NL模板: "查询{DATE_RANGE}销售额超过{THRESHOLD}的{CITY}客户"
SQL模板: SELECT ... WHERE city={CITY} AND date BETWEEN {DATE_START} AND {DATE_END} 
         GROUP BY ... HAVING SUM(amount) > {THRESHOLD}
```

### 2.3 SQL结构指纹设计

```python
@dataclass
class SQLStructureFingerprint:
    """SQL结构指纹：抽象后的骨架特征"""
    
    # 基础结构
    tables_involved: List[str]          # 涉及的表
    join_pairs: List[Tuple[str, str]]   # JOIN关系
    join_types: List[str]               # JOIN类型
    
    # SELECT特征
    aggregation_functions: List[str]    # COUNT/SUM/AVG/MAX/MIN
    has_distinct: bool
    
    # WHERE特征
    where_column_ops: List[Tuple[str, str]]  # (column, operator)
    logical_connectors: List[str]       # AND/OR结构
    has_subquery_in_where: bool
    
    # GROUP BY / ORDER BY
    group_by_columns: List[str]
    order_by_columns: List[Tuple[str, str]]  # (col, ASC/DESC)
    
    # 高级特征
    has_window_function: bool
    subquery_count: int
    nesting_depth: int
    
    def to_hash(self) -> str:
        """生成结构哈希，用于精确匹配"""
        
    def to_feature_vector(self) -> List[float]:
        """转为向量，用于结构相似度计算"""
```

## 三、案例匹配机制

### 3.1 匹配流程

```
Phase 1: 查询预处理
  ┌──────────────────────────────────────────┐
  │  实体识别 → 模板抽象 → 意图分类           │
  │  "销售额top10" → "{METRIC}{RANKING}"     │
  └──────────────────────────────────────────┘

Phase 2: 多路召回
  ┌────────────┬────────────┬──────────────┐
  │向量语义召回 │模板结构召回 │关键词召回     │
  │(Top-K=50)  │(Top-K=30)  │(Top-K=20)    │
  └────────────┴────────────┴──────────────┘
         合并去重 → 候选集(~60-80条)

Phase 3: 精排匹配
  ┌────────────┬────────────┬──────────────┐
  │NL语义精排   │SQL结构对比 │负例区分       │
  │Cross-Encoder│AST diff   │Contrastive   │
  └────────────┴────────────┴──────────────┘

Phase 4: 输出
  ┌──────────────────────────────────────────┐
  │ 案例适配 → SQL生成 → 执行验证             │
  └──────────────────────────────────────────┘
```

### 3.2 相似度计算

**多维度相似度融合**：

```python
def compute_similarity(query, case):
    # 1. NL语义相似度
    nl_sim = cosine_similarity(query.nl_embedding, case.nl_embedding)
    
    # 2. 模板结构相似度
    template_sim = cosine_similarity(query.nl_template_embedding, 
                                     case.nl_template_embedding)
    
    # 3. SQL结构相似度
    structure_sim = euclidean_similarity(
        query.sql_fingerprint.to_feature_vector(),
        case.sql_fingerprint.to_feature_vector()
    )
    
    # 4. 意图一致性
    intent_sim = jaccard_similarity(query.query_intents, case.query_intents)
    
    # 5. Schema覆盖度
    schema_sim = len(set(query.tables) & set(case.tables)) / len(query.tables)
    
    # 加权融合
    final_sim = 0.3 * nl_sim + 0.25 * template_sim + 0.2 * structure_sim 
              + 0.15 * intent_sim + 0.1 * schema_sim
    
    return final_sim
```

### 3.3 负例区分机制（核心创新）

**问题**："销售额top10" vs "销售额bottom10" 语义极近但逻辑相反

**解决方案**：

```python
def contrastive_check(query, candidate_case):
    """对比检查：区分相似但不同的查询"""
    
    # 1. 检查负例关联
    if candidate_case.case_id in query.confusable_case_ids:
        # 触发深度对比
        return deep_contrastive_analysis(query, candidate_case)
    
    # 2. 关键语义差异检测
    critical_diffs = detect_critical_differences(query, candidate_case)
    # 检测项：排序方向、聚合类型、时间范围方向
    
    # 3. 结构差异验证
    if candidate_case.confusion_notes:
        for key, note in candidate_case.confusion_notes.items():
            if applies_to(query, key):
                return False, note
    
    return True, "匹配通过"
```

**关键差异检测项**：

| 检测项 | 相似但不同示例 | 检测方法 |
|--------|---------------|---------|
| **排序方向** | "top10" vs "bottom10" | ORDER BY方向检查 |
| **聚合类型** | "总量" vs "平均值" | 聚合函数检查(SUM vs AVG) |
| **时间方向** | "最近7天" vs "过去7天" | 时间边界语义分析 |
| **比较方向** | "超过100万" vs "低于100万" | WHERE操作符检查(GT vs LT) |
| **存在性** | "有没有" vs "有多少" | 查询意图检查(EXISTS vs COUNT) |

## 四、案例召回策略

### 4.1 多路召回设计

|召回通道|召回方式|Top-K|用途|
|-------|-------|-----|----|
|**向量召回**|Embedding相似度+ANN索引|50|语义相近候选|
|**结构召回**|SQL结构哈希+倒排索引|30|相同结构模板|
|**关键词召回**|BM25+Schema关键词|20|表名/列名命中|

**合并去重策略**：
```python
def merge_recall_results(vector_hits, structure_hits, keyword_hits):
    # 优先级：结构命中 > 向量命中 > 关键词命中
    candidates = {}
    
    # 结构命中（高优先级）
    for case in structure_hits:
        candidates[case.case_id] = (case, 'structure', 0.9)
    
    # 向量命中
    for case in vector_hits:
        if case.case_id not in candidates:
            candidates[case.case_id] = (case, 'vector', 0.8)
    
    # 关键词命中（低优先级）
    for case in keyword_hits:
        if case.case_id not in candidates:
            candidates[case.case_id] = (case, 'keyword', 0.6)
    
    return list(candidates.values())
```

### 4.2 阈值过滤策略

```python
THRESHOLD_CONFIG = {
    'exact_match': 0.95,       # 精确匹配 → Layer 1
    'template_match': 0.85,    # 模板匹配 → Layer 2
    'fallback': 0.70,          # 阈值以下 → Layer 3
}

def route_by_similarity(similarity_score):
    if similarity_score >= THRESHOLD_CONFIG['exact_match']:
        return 'Layer 1: 直接复用'
    elif similarity_score >= THRESHOLD_CONFIG['template_match']:
        return 'Layer 2: 模板实例化'
    else:
        return 'Layer 3: LLM生成'
```

### 4.3 召回失败处理

|失败类型|处理策略|
|-------|-------|
|**无结构匹配**|降级到向量召回，扩大候选集|
|**候选集过小(<5)**|触发LLM生成，成功后入库新案例|
|**负例触发**|标注混淆关系，记录失败原因|
|**Schema不匹配**|过滤无效候选，补充Schema信息|

## 五、创新点技术实现

### I1: 自动质量保证机制

**多维度自动验证**：

```python
class CaseQualityValidator:
    """案例入库质量验证器"""
    
    def validate(self, sql, result, context):
        checks = []
        
        # 1. 执行验证
        checks.append(('exec_success', self.check_execution(sql)))
        
        # 2. 结果合理性验证
        checks.append(('result_reasonable', self.check_result_reasonable(result)))
        # 检查项：count不能为负、时间范围合理、数值范围合理
        
        # 3. 一致性验证
        checks.append(('consistency', self.check_consistency(sql, context)))
        # 同一模板的历史案例结果是否一致
        
        # 4. 对抗验证
        checks.append(('robustness', self.check_robustness(sql, context)))
        # 微扰问题是否得到一致结果
        
        # 计算质量分
        quality_score = sum(v for _, v in checks) / len(checks)
        
        return quality_score >= QUALITY_THRESHOLD, checks
```

### I3: 退化检测机制

```python
class CaseDegradationDetector:
    """案例库退化检测器"""
    
    def detect_degradation(self, case, current_schema):
        # 1. Schema变更检测
        if self.schema_changed(case.schema_subset, current_schema):
            return 'schema_outdated', case.case_id
        
        # 2. 表/列失效检测
        invalid_refs = self.check_invalid_refs(case, current_schema)
        if invalid_refs:
            return 'refs_invalid', invalid_refs
        
        # 3. 使用率异常检测
        if case.hit_count > 0 and case.success_count / case.hit_count < 0.5:
            return 'low_success_rate', case.case_id
        
        return None
```
